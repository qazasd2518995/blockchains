## 📌 第一部分：加密博彩遊戲基礎概念

### 什麼是加密博彩遊戲（Crypto Casino Games）？

加密博彩遊戲是以加密貨幣（USDT、BTC、ETH 等）作為投注與結算媒介的線上博彩遊戲，是傳統線上娛樂場的演進版本。

### 核心特徵對比

| 面向 | 傳統博彩 | 加密博彩 |
|------|---------|---------|
| **投注貨幣** | 法幣 | USDT/BTC/ETH |
| **存提款速度** | 1-5 天 | 數分鐘鏈上到帳 |
| **公平性驗證** | 信任第三方牌照 | Provably Fair 可自行驗證 |
| **遊戲節奏** | 長、慢 | 短局 5-30 秒 |
| **最低投注** | $0.1-$1 | 可低至 $0.01 |

### 三大技術特色

1. **Provably Fair（可驗證公平）**
   Server Seed + Client Seed + Nonce 雜湊機制，讓玩家能事後驗證結果沒被竄改。

2. **短局高頻**
   多在 5-30 秒內完局，玩家每小時可下數百注。

3. **高 RTP**
   通常 96%-99%（比傳統老虎機的 92%-94% 高）。

### 常見別稱

- **Crash Games**（崩盤類遊戲）
- **Instant Games**（即時遊戲）
- **Mini Games / Originals / Turbo Games**

## 🔐 第六部分：Provably Fair 可驗證公平機制

### 為什麼需要？

傳統線上博彩玩家只能「相信」莊家，加密博彩用密碼學建立**信任最小化**。

### 三大元素

1. **Server Seed（伺服器種子）**
   莊家開局前生成的隨機字串，開局前公布其 SHA-256 雜湊值（承諾這串雜湊代表一組不可竄改的種子）。

2. **Client Seed（客戶端種子）**
   玩家瀏覽器生成或玩家自訂，可隨時換。

3. **Nonce（計數器）**
   同一對 seed 下第 N 局。

### 驗證流程

```
1. 開局前：莊家公布 SHA-256(Server Seed)
2. 下注 → 系統 HMAC_SHA256(Server Seed, Client Seed + ":" + Nonce)
         → 轉換為倍率/點數/落點
3. 換 Seed 或結束時：莊家公開 Server Seed 原文
4. 玩家自行驗證：
   - SHA-256(原文) 是否等於開局前公布的雜湊 → 確認 seed 沒被換掉
   - 用相同算法重算結果 → 確認莊家沒動手腳
```

### 各遊戲的驗證方式

| 遊戲類型 | 演算法 |
|---------|--------|
| **Crash**（Aviator/JetX） | 指數分布公式生成崩盤點 |
| **Dice** | 取 hex 前 5 位 mod 10000 ÷ 100 = 0.00-99.99 |
| **Plinko** | 每排取一個 bit 決定往左或往右 |
| **Mines** | 隨機排列 25 個位置，選前 N 個為地雷 |

### Provably Fair **不保證**的

- ❌ 不保證你會贏：RTP 仍是莊家優勢的上限
- ❌ 不保證平台會付款：只驗結果公正
- ❌ 理論剩餘風險：莊家可選擇偏向某些結果的 seed（但機率極低）
